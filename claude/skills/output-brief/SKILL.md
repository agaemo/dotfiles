# output-brief

```
SKILL_DIR = このファイル（SKILL.md）を Read したときのファイルパスからファイル名を除いたディレクトリ部分
```

アウトプットを伴うタスクを「データ確認 → 方針確認 → 承認 → 出力」の流れで実施する。

## 発動判断

以下のいずれかに該当する場合のみブリーフィングを行う。**全て指定済みなら即出力する（ステップ1〜3をスキップしてステップ4へ進む）。**

- 保存先（ファイル保存 / チャット表示）が未指定
- 出力形式（HTML / Markdown / CSV …）が未指定
- HTML でレイアウト・カラーが未指定
- コンテンツ内容・構成が不明

## フロー

### 1. データ確認

指示から以下を把握する。不明な点があれば **1 回でまとめて** 質問する。

- 何を作るか（ファイル種別・内容）
- データ・素材はあるか（URL・ファイルパス・テキスト）
- 保存先の指定はあるか（ファイルとして保存 / チャット表示）

### 2. ブリーフィング

選択肢は **`AskUserQuestion` ツールで提示する**（テキストで羅列しない）。

#### HTML の場合

`AskUserQuestion` を 1 回で、レイアウトとカラーを同時に提示して選んでもらう。

**レイアウト**
- `simple-list` — 縦一覧。テキスト中心・情報量が多いコンテンツ向き
- `panel` — カードグリッド。同程度の重要度の項目を視覚的に並べる向き

**カラーテーマ**
- `warm` — オレンジ・ベージュ。温かみがある
- `cool` — パープル・ラベンダー。落ち着いた印象
- `blue` — 青系。清潔感・信頼感
- `green` — 緑系。自然・安心感
- `dark` — 暗い背景。モダン・集中感
- `light` — 白ベース。シンプル・クリーン

サンプルは `$SKILL_DIR/assets/samples/<layout>-<color>.html` に保存されている。
ブラウザで確認できることをユーザーに伝える。

#### Markdown の場合

`AskUserQuestion` で以下を確認する:

- **見出し構造**: フラット（H2 のみ）/ 階層あり（H2 + H3）
- **テーブル**: あり / なし
- **コードブロック**: あり / なし

#### CSV の場合

`AskUserQuestion` で以下を確認する:

- **区切り文字**: カンマ / タブ
- **ヘッダ行**: あり / なし
- **文字コード**: UTF-8 / Shift-JIS

#### JSON の場合

`AskUserQuestion` で以下を確認する:

- **構造**: 配列 / オブジェクト（キーあり）
- **インデント**: あり（2 スペース）/ なし（minified）

#### その他のフォーマット

形式・主要構成（セクション・項目）を列挙して確認する。不明点は 1 回でまとめて質問する。

### 3. 承認・修正

WAIT_FOR: ユーザーの承認（「OK」「はい」「進めて」等）を受けてからステップ4へ進む。
PROHIBITED: 承認なしにステップ4（出力・保存）を実行すること。
修正を求められた場合はステップ2に戻って再提示する（最大3ラウンド。収束しない場合は中断してユーザーに方向性を確認する）。

### 4. 出力

ステップ2で確認した形式・レイアウト・カラー等の選択を参照してアウトプットを生成する。

- ファイル保存の場合: Write ツールで指定パスに保存し、ファイルの存在を確認してからパスを報告する
- チャット表示の場合: コードブロックで出力する
- 保存に失敗した場合: エラーを報告し、別のパスをユーザーに確認する
