# output-brief

アウトプットを伴うタスクを「方針確認 → 承認 → 出力」の流れで実施する。

## フロー

### 1. タスク理解

指示から以下を把握する:
- 何を作るか（ファイル種別・内容）
- データ・素材はあるか
- 保存先の指定はあるか

### 2. ブリーフィング

#### HTML の場合

以下の選択肢を提示し、組み合わせをユーザーに選んでもらう。
サンプルは `assets/samples/<layout>-<color>.html` を開いて確認できることを伝える。
パターンの詳細は `README.md` を参照。

**レイアウト**
- `simple-list` — シンプルな縦一覧
- `panel` — カード（パネル）グリッド

**カラーテーマ**
- `warm` — 暖色系（オレンジ・ベージュ）
- `cool` — 寒色系（パープル・ラベンダー）
- `blue` — 青系
- `green` — 緑系
- `dark` — ダーク
- `light` — ライト

提示例:
```
レイアウト: simple-list / panel
カラー: warm / cool / blue / green / dark / light
サンプル確認: assets/samples/ 内の HTML をブラウザで開いてください。
```

#### 他フォーマットの場合

形式と主要構成（セクション・項目）を列挙して確認する。
不明点があれば 1 回でまとめて質問する。

### 3. 承認・修正

ユーザーが承認したら着手する。修正を求められた場合は修正して再確認する。

### 4. 出力

承認された方針に基づいてアウトプットを生成・保存し、パスを報告する。
