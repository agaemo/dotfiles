01 課題

指示だけでは、期待と食い違う

  • 保存先・出力形式が曖昧なまま生成し、やり直しになる
  • HTML のレイアウト・配色は好みが分かれ、後戻りが多い
  • 調査が必要な内容ほど、思い込みで進めがち

02 解決策

合意してから出力する、3ステップ

1
データ確認

何を・何のデータで・どこに出力するか把握する

→
2
ブリーフィング

形式・レイアウト・カラーを番号リストで選んでもらう

→
3
承認 → 出力

要約を提示し、承認を得てから生成・保存する

03 対応フォーマット

形式ごとに聞くことを変える

HTML

レイアウト（4種）→ カラー（6種）の2段階で選択

Markdown

見出し構造・テーブル・コードブロックの有無を確認

CSV

区切り文字・ヘッダ行・文字コードを確認

JSON

配列 / オブジェクト構造とインデントを確認

04 効果

手戻りを減らし、選べる安心感を

3ラウンドまでの修正で収束させる設計
承認なしに出力しない。これだけで、作り直しの手間がほぼ無くなった。
