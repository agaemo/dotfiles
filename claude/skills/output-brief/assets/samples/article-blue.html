    Claude Code スキル設計ガイド

    設計スキルサンプルドキュメント — output-brief / article レイアウト

    概要

    Claude Code スキルは、繰り返し使うワークフローを SKILL.md ファイルとして定義し、/skill-name で呼び出せる仕組みです。スキルを設計するときは「コールドスタートのエージェントが迷わず実行できるか」を基準にします。

    スキルは Claude への指示書であり、人間向けのドキュメントではない。シンプルで一貫性のある構造が、品質の高い出力につながる。

    スキルの種類

    プログラム型

    Agent ツールでサブエージェントを呼び出す構造。複雑なタスクを並列・分散して処理できます。

    • 親エージェントがオーケストレーション
    • サブエージェントが実作業を担当
    • 完了待ちを WAIT_FOR で明示する

    指示文型

    Claude への自然言語指示のみで構成。ユーザーとのインタラクションが中心のスキルに向いています。スコープが小さく、判断の分岐が少ない場合に最適です。

    設計原則

    以下の3点を常に確認してください。

    1
    Progressive Disclosure

    description だけでトリガー判断できること

    2
    ゲート設計

    不可逆操作の前に必ず WAIT_FOR を置く

    3
    1回でまとめて

    ユーザーへの質問は1メッセージに集約する

    実装例

    WAIT_FOR: ユーザーの承認を受けてから次へ進む
PROHIBITED: 承認なしにファイルを書き換えること

    上記のように WAIT_FOR と PROHIBITED を対で使うと、ゲートのスキップを防げます。スキルの品質はゲート設計の厳密さで決まると言っても過言ではありません。

    用語集

    WAIT_FOR — この行が現れた直後でユーザー入力を待つ。承認なしに次のステップへ進んではならない。

    PROHIBITED — 実行を禁止するアクションを明示する。モデルが勝手に実行しないよう強調する目的で使う。

    SKILL_DIR — スキルファイルが置かれているディレクトリの絶対パス。ファイルの読み書きに利用する。
