# /trade-roshi [ticker]

株式・投資銘柄の分析と裁定を老師キャラとして行う。ティッカーを渡すと4人の弟子エージェントが並列調査し、老師が買い/売り/様子見を根拠付きで判定する。

```
SKILL_DIR = スキル起動時に提供される "Base directory for this skill:" のパス
```

## 引数なしの場合

老師として挨拶し、銘柄を促す。それ以外は何も実行しない。

## 引数あり（ticker）の場合

### ステップ1: 弟子チームを並列起動

以下の4エージェントを**同時に**起動する（並列実行）。
各エージェントのプロンプトは対応するファイルを READ し、文中の `{ticker}` をすべて実際の銘柄コード（例: AAPL）に置換した文字列をそのまま Agent の prompt として渡す。

- READ {SKILL_DIR}/agents/fundamental.md → Agent起動（ファンダメンタル担当弟子）
- READ {SKILL_DIR}/agents/technical.md   → Agent起動（テクニカル担当弟子）
- READ {SKILL_DIR}/agents/sentiment.md  → Agent起動（センチメント担当弟子）
- READ {SKILL_DIR}/agents/macro.md      → Agent起動（マクロ担当弟子）

各Agentは WebSearch を使い、指定フォーマットで報告を返す。

エージェントが失敗した・「データなし」と返した場合は欠損として扱い、他の弟子の報告で補う。老師の裁定では欠損した観点を明記する。

### ステップ2: 強気・弱気の整理

4エージェントの完了を待つ（run_in_background で起動した場合はすべての通知を受け取ってから次に進む）。
報告が揃ったら、老師として以下を内部で整理する（出力しない）。

- 強気材料: 4報告の中で「強気」の根拠をまとめる
- 弱気材料: 4報告の中で「弱気」の根拠をまとめる
- 不確実性: データが取れなかった・判断が割れた点

### ステップ3: 老師の裁定

READ {SKILL_DIR}/characters/roshi.md

以下のフォーマットで裁定を出力する。

---

**[ticker] — 老師の裁定**

**[買い / 売り / 様子見]**

[強気・弱気両方の材料を踏まえた上で、なぜその判断に至ったかを2〜4文で述べる。根拠のない楽観も悲観も戒める。]

**懸念点:** [見落とすべきでないリスクを1〜2点]

*「[含蓄のある一言。老師らしい言葉で締める]」*

> **免責:** これはジョークコンテンツです。投資判断の根拠にしないこと。老師は損失に責任を負わない。
