# プレビュー生成（共通ルール）

craft の new-project・new-app フロー（agent-chain.md）・new-static フローの各GATE（design-brief・plan.md・統合設計書）で、承認前にローカルHTMLでプレビューを生成する際の共通手順。メインClaudeが行う。呼び出し元のSTEPからは、このファイルをプレビュー生成チェーン中に未読の場合のみ読み込み（セッションが変わった場合は再度READする）、対象ファイル・図解内容の差分のみを実行すること。

## Artifact を使わない理由

Artifact は外部ホストへのリクエストが厳格にブロックされる（CSP制約）ため使用しない。ローカルHTMLファイルとして出力する。

## CDN（外部ライブラリ）の利用方針

- 装飾目的（Webフォント等、見た目の向上のみに関わるもの）でのCDN利用は許容する
- 内容の理解に必須な要素（図解・データ表示・依存関係の可視化等）は、閲覧時にオフラインだと
  描画されなくなるリスクを避けるため、外部CDN（mermaid.js等）に頼らず自作のSVG/CSSで完結させること

## 手順

呼び出し元STEPが示す「出力先: <パス>」の値を、以下の `<出力パス>` として使う。
対象ファイルが複数（例: STEP6の3文書統合）の場合も、呼び出し元の指示に従い単一のHTMLにまとめる。

```
Skill ツールで artifact-design を読み込んでから作成する
<対象Markdownファイル（複数の場合は呼び出し元の指示に従い1枚に統合）> の内容から確認用HTMLを作成する
  [各STEP固有の図解内容・レイアウトは呼び出し元STEPの指示に従う]
WRITE <出力パス>
ASSERT EXISTS(<出力パス>)
IF NOT EXISTS:
  REPORT: "プレビューHTMLの生成に失敗しました。内容確認は元のMarkdownファイルで行いますか、再試行しますか？"
  WAIT_FOR: ユーザーの判断
  IF 再試行: 最初からやり直す
  ELSE: 元のMarkdownファイルのパスを伝えて次に進む（プレビュー確認はスキップ）
ELSE:
  <出力パス> のパスをユーザーに伝える。
  ローカル環境でブラウザを自動起動できる場合（例: macOSの `open` コマンド）はメインClaudeが実行し、
  できない場合はパスのみ伝えてユーザー自身に開いてもらう
ENDIF
```
