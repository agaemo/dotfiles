# consult PR テンプレート

PR 作成時の本文フォーマット。`ISSUE_NUMBER` は issue を作成した場合の番号、なければ省略する。

```
## 背景・課題
[相談内容と根本原因。なぜこの変更が必要だったかを知らない人が読んでも理解できる粒度で書く]

## 検討した選択肢
[採用した案だけでなく、却下した案とその理由も記載する]

## 決定内容と理由
[何を選び、なぜそれを選んだか。制約・リスク・トレードオフを含める]

## 変更内容
### フェーズ 1: [名称]
- 変更ファイル: [一覧]
- 変更概要: [何をどう変えたか]
- 検証結果: [実行したコマンドと結果]

### フェーズ N: ...（フェーズ数だけ繰り返す）

## 削除したもの
- 旧コード・依存パッケージ: [一覧]

## 追加したもの
- 新しい依存パッケージ: [名前・バージョン・用途]

## 残作業（マージ後に手動で対応が必要なもの）
- [ ] [環境変数の更新など、具体的なタスクをチェックリスト形式で]

## 確認方法
[レビュアーがローカルで動作確認するための手順]

Closes #ISSUE_NUMBER
```
