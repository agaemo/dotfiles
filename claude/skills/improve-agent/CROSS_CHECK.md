# 親子エージェント一貫性チェック（CROSS_MODE = true のみ）

```
REQUIRE: MULTI_FILE_PROCESS の全ファイル処理（ステップ2〜4）完了後に実行する
NOTE: 各ファイルの EXTRACT 結果をそのまま使う。ファイルの再読みは行わない

COMPARE 親エージェント vs 各子エージェント:

[X1] 引き継ぎインターフェース
  CHECK: 親エージェントが子エージェントを呼び出す際に渡すプロンプト・パラメータと、
         子エージェントが期待する入力（引数・前提条件・コンテキスト）が一致しているか

[X2] 役割境界の重複
  CHECK: 複数エージェント間で同じ処理（ステップ・判断ロジック）を二重定義していないか
         （判断基準: そのステップが両エージェントに存在する場合、どちらかを削除しても目的が達成できるなら重複）

[X3] tools の整合性
  CHECK: 子エージェントが実行する処理に必要なツールが、子エージェントの tools フィールドに含まれているか
  CHECK: 親が子エージェントへ期待する出力（ファイル・構造化データ等）を子が実際に生成できるか

[X4] エラー・異常系の引き継ぎ
  CHECK: 子エージェントが失敗・中断した場合の処理が親エージェント側に定義されているか

GENERATE 改善提案（不一致がある場合のみ）:
  分類: [X1〜X4]
  優先度: HIGH（動作に影響）/ MEDIUM（品質に影響）
  場所: 親エージェント or 子エージェント（修正すべき側）
  問題: <不一致の内容>
  推奨: <修正案>
  理由: <1行>

不一致がない場合: "親子エージェント一貫性: 問題なし" とのみ報告する
```
