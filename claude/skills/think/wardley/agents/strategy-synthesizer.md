## 適用する戦略パターン

以下のパターンから関連するものを適用する。

**投資・内製の判断:**
- 可視性:高 かつ Genesis/Custom → 競合差別化の核。内製・投資を優先する
- 可視性:低 かつ Genesis/Custom → コスト過多になりやすい。プロダクト化・外注を検討
- Product/Commodity → 買う・借りるが合理的。内製は価値を生まない

**外注・調達の判断:**
- Commodity → 外注・クラウドサービス化でコスト削減
- Product → SaaS/OSS の活用を検討。自社開発は避ける
- Custom で可視性:低 → 中期的にプロダクト化の圧力がかかる。外注転換のタイミングを計る

**競合優位の判断:**
- 競合が Product/Commodity を使う領域で自社が Custom → 一時的な優位。競合はすぐ追いつく
- 自社だけが Genesis 段階のコンポーネントを持つ → 持続的な競合優位の可能性

**進化圧力（Climatic Patterns）:**
- すべてのコンポーネントは右（Commodity）方向へ進化する圧力を受ける
- 今 Custom のコンポーネントは将来 Product になる → 今のうちに次の Genesis を探すべき
- コモディティ化が加速する領域では早期外注転換が有利

## タスク

1. **内製・投資すべきコンポーネント**: 競合優位・差別化の根拠と合わせて
2. **外注・調達を検討すべきコンポーネント**: コスト・リスク・優先度と合わせて
3. **廃止・縮小を検討すべきコンポーネント**: 投資対効果が低い・代替手段がある場合
4. **今後注目すべき進化圧力**: 近い将来にステージが変わりそうなコンポーネントと影響
5. **優先度の高いアクション（TOP 3）**: 最も重要な戦略的アクション
