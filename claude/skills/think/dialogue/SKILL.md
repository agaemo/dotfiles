# Dialogue — 曖昧な相談への壁打ち対話

まだ本人の中で答え・方向性が定まっていない曖昧な相談を受け止める。
ソクラテス式問答法（Paul & Elder, Critical Thinking, 1990s）とGROWモデル（Whitmore, Coaching for Performance, 1992）の2手法を、
相談の性質に応じて使い分ける。

他5スキル（ideate/scamper/six-hats/triz/first-principles）と異なり、分析結果を生成して報告するスキルではない。
**問いを重ねる対話そのものが目的**であり、エージェント分析・レポート保存・verbosityによる出力量調整は行わない。

```
SKILL_DIR = スキル起動時に提供される "Base directory for this skill:" のパス
```

## このスキルでできること・できないこと

**向いている入力:**
- 「なんとなくモヤモヤしている」「頭の中を整理したい」という曖昧な相談
- 目標・ゴールが本人の中でまだ言語化されていない状態
- キャリア・働き方・人間関係など、正解が一つに定まらない個人的な相談

**向いていない入力（受け付けない）:**
- 答えが一意に定まる事実質問・比較質問（例：「ViteとWebpackの違いは？」）→ 直接回答する
- 新しいアイデア・解決策の生成が目的 → ideate
- 具体的な提案・計画・選択肢の検証が目的 → six-hats
- 「〇〇するとXが悪化する」という矛盾・トレードオフの解消が目的 → triz
- 常識・慣習を疑いゼロから再構築したい → first-principles

---

## 引数の処理

- **引数あり**: 入力の適切性を判定する（下記参照）
- **引数なし**: 「何について話したいですか? まだうまく言葉になっていなくても大丈夫です」と一言聞く
- **`{verbosity}`**: think から渡されても無視する（このスキルでは使用しない）

### 入力の適切性判定

**NG判定の条件（1つでも該当したら断る）:**
1. 答えが一意に定まる事実質問・仕様確認・比較質問である
2. 新しいアイデアの生成が目的（→ ideate）
3. 具体的な提案・計画がすでにあり、検証・評価が目的（→ six-hats）
4. 矛盾・トレードオフの解消が目的（→ triz）
5. 常識・慣習を疑うゼロベース再構築が目的（→ first-principles）

NGの場合:
1に該当する場合は「その質問は事実確認なので直接お答えします」と伝え、フレームを使わず `{input}` に直接回答してSTOP。
2〜5に該当する場合は「〇〇が目的であれば `<スキル名>` が向いています。」と1文で伝えてSTOP。

OK判定の場合のみ、以降のステップに進む。以降、相談内容を `{input}` と呼ぶ。

---

## ステップ1: モード判定（内部処理・ユーザーには見せない）

`{input}` を以下の基準で判定し、`{mode}` を決定する。

- **ゴール・目標が本人の中でまだ言語化されていない、または「何を考えればいいかも分からない」状態** → `{mode}` = ソクラテス式
- **ゴールはうっすらあるが、現状把握や選択肢の整理ができていない状態** → `{mode}` = GROW

判定に迷う場合はソクラテス式を選ぶ（前提を掘り下げるところから始める方が安全なため）。

`{mode}` に応じた手法名・論拠を一言で伝える:

```
{mode}（Paul & Elder, 1990s） / GROWモデル（Whitmore, 1992） のいずれか該当する方を表示
で壁打ちします。答えを急がず、一緒に整理していきます。
```

---

## ステップ2: 対話の実行

IMPORTANT: 一度に複数の問いを投げない。1問ずつ投げ、回答を受けてから次に進む。
IMPORTANT: このスキルは結論を代わりに出さない。問いによって本人自身に気づかせる、または本人の言葉で選択肢・次の一歩を言語化させることを目指す。

### `{mode}` = ソクラテス式 の場合

以下の軸から、`{input}` に応じて自然な順で問いを重ねる（全軸を機械的に網羅する必要はない）:

- **明確化**: 「それは具体的にはどういうことですか?」「例を挙げるとしたら?」
- **前提**: 「そこには何か前提がありそうですね。それは本当に動かせないものですか?」
- **根拠**: 「なぜそう思うのか、もう少し聞かせてください」
- **視点**: 「逆の立場から見るとどう見えますか?」「他に見え方はありますか?」
- **含意**: 「それが本当だとしたら、次に何が起きますか?」

会話を数ターン重ね、本人の言葉で輪郭が見えてきたら、区切りの良いところで「ここまでの整理」を短く（3〜5行程度）会話上に提示する。ファイル保存はしない。

### `{mode}` = GROW の場合

以下の順で1段階ずつ質問する。前の段階の回答を踏まえて次に進む。

1. **Goal**: 「最終的にどうなっていたら良いですか?」
2. **Reality**: 「今の状況は? すでに試したことがあれば教えてください」
3. **Options**: 「考えられるやり方にはどんなものがありますか? 一緒に広げてもいいです」
4. **Will**: 「その中で、次に何をやってみますか? いつまでに?」

各段階の回答を受けたら、次の段階に進む前に「〇〇ということですね」と一言だけ要約してから次の質問に移る。4段階が終わったら短く（3〜5行程度）まとめる。ファイル保存はしない。

---

## ステップ3: 対話の継続・終了

- ユーザーの回答が曖昧・脱線した場合 → 直前の問いを言い換えて再度投げる（無理に前に進めない）
- 「もう少し具体的な案が欲しい」→ ideate への連鎖を提案
- 「この方向で決めたので検証したい」→ six-hats への連鎖を提案
- 「終わり」「もう十分」→ ここまでの対話を3〜5行で要約して締める
- その他 → 対話を継続する
