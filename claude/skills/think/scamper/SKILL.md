# SCAMPER — 発想変形・発展法

Alex Osborn「Applied Imagination」(1953) の発想法を Bob Eberle (1971) が体系化。
既存のアイデア・製品・サービス・プロセスを7軸で強制変形し、新たなアイデアを引き出す。

```
SKILL_DIR = スキル起動時に提供される "Base directory for this skill:" のパス
```

## このスキルでできること・できないこと

**向いている入力:**
- 既存の製品・サービス・ビジネスモデル・プロセスの改良・変形
- 「もっと良くしたい」「別の展開はないか」という発展探索
- 競合との差別化アイデアの発掘

**向いていない入力（受け付けない）:**
- 変形する「既存の対象」が何もない完全ゼロの発想（→ first-principles が向いている）
- 「良いか悪いか」の評価・検証（→ six-hats が向いている）
- 相反する要件の解消（→ triz が向いている）

---

## 引数の処理

- **引数あり**: 入力の適切性を判定する（下記参照）
- **引数なし**: 「SCAMPER で変形・発展させたい対象は何ですか?」と一言聞く

### 入力の適切性判定

**NG判定の条件（1つでも該当したら断る）:**
1. 変形する既存の対象が何も含まれていない
2. 「良いか悪いか」の評価・比較が目的
3. 完全ゼロから概念を生み出すことを求めている

NGの場合:
```
SCAMPER は「既存のアイデア・製品・プロセス」を変形・発展させる手法です。
今の入力「{input}」には変形する対象が明確でないため、SCAMPER での分析には向いていません。

変形する対象を具体的に教えてください。例：
- 「自社の○○サービス」
- 「現在の△△業務フロー」
- 「競合が使っている□□という仕組み」
```

OK判定の場合のみ以降に進む。以降、分析対象を `{input}` と呼ぶ。

## ステップ1: 4エージェントを並列起動

以下のファイルを READ し、`{input}` を置換したプロンプトとして `run_in_background: true` で**同時に** Agent 起動する。全エージェントの完了通知を受け取ってからステップ2に進む。

- {SKILL_DIR}/agents/substitute-combine.md を READ し、ファイル内の `{input}` を分析対象テキストに置換したプロンプトとして Agent を起動する（S・C軸、run_in_background: true）
- {SKILL_DIR}/agents/adapt-modify.md を READ し、ファイル内の `{input}` を分析対象テキストに置換したプロンプトとして Agent を起動する（A・M軸、run_in_background: true）
- {SKILL_DIR}/agents/other-uses-eliminate.md を READ し、ファイル内の `{input}` を分析対象テキストに置換したプロンプトとして Agent を起動する（P・E軸、run_in_background: true）
- {SKILL_DIR}/agents/reverse.md を READ し、ファイル内の `{input}` を分析対象テキストに置換したプロンプトとして Agent を起動する（R軸、run_in_background: true）

エージェントが失敗した場合は「報告なし」として扱い、統合で欠損を明記する。

---

## ステップ2: 統合・ファイル出力

4エージェントの結果を受け取り、以下の内容を組み立てる。

PROHIBITED: ユーザーの保存先確認なしにファイルを書くこと

ファイルを保存する前に「保存先のパスを教えてください（例: ~/Desktop）」と一言聞く。ユーザーが答えたパスに対し:

1. Bash で `date +%Y%m%d-%H%M` を実行してタイムスタンプを、`date +%Y-%m-%d` を実行して日付をそれぞれ取得する。ファイル名は `scamper-<YYYYMMdd-HHmm>.md` とする
2. 指定パスが存在しない場合は Bash で `mkdir -p <path>` を実行してから保存する
3. Write ツールで保存する

パスが省略されたり「ここ」「カレント」と言われた場合はカレントディレクトリに保存する。

---

ファイルの内容:

````markdown
# SCAMPER 発想レポート
{input の冒頭30字程度} — {YYYY-MM-DD}

## サマリー

| 軸 | 代表アイデア |

| S 代替 | [代表的なアイデア1〜2点] |
| C 結合 | [代表的なアイデア1〜2点] |
| A 適応 | [代表的なアイデア1〜2点] |
| M 変更 | [代表的なアイデア1〜2点] |
| P 転用 | [代表的なアイデア1〜2点] |

| R 逆転 | [代表的なアイデア1〜2点] |
````

---

ファイル保存後、会話上には「`{実際の保存フルパス}` に保存しました」と一言伝え、サマリー表と注目アイデア TOP3 のみ表示する。詳細セクションは会話上には出力しない。

## ステップ3: 対話

- 「S・Cをもっと詳しく」→ substitute-combine.md を `{input}` 置換して再起動し、補足を会話上のみ出力
- 「A・Mをもっと詳しく」→ adapt-modify.md を `{input}` 置換して再起動し、補足を会話上のみ出力
- 「P・Eをもっと詳しく」→ other-uses-eliminate.md を `{input}` 置換して再起動し、補足を会話上のみ出力
- 「R・逆転をもっと詳しく」→ reverse.md を `{input}` 置換して再起動し、補足を会話上のみ出力
- 「このアイデアのリスクは?」→ six-hats での検証を提案
- 「終わり」→ 簡潔に締める
- その他 → 統合した視点で答える
