# think — 思考・分析オーケストレーター

思考・分析・検討の相談を受け付けるオーケストレーター。
「〜を分析したい」「〜について考えたい」「six-hats で検証したい」のような
入力に対して適切なスキルへ委譲する。

```
SKILL_DIR = スキル起動時に提供される "Base directory for this skill:" のパス
```

## 使えるスキル

| スキル | 得意な入力 |
|-------|-----------|
| six-hats | 具体的な提案・計画・選択肢の多角的検証（Six Thinking Hats） |
| scamper | 既存のアイデア・製品・プロセスを7軸で変形・発展させたい（SCAMPER法） |
| first-principles | 常識・慣習を疑い、ゼロから再構築したい（第一原理思考） |
| triz | 相反する要件を同時に満たす解法を探したい（TRIZ・40発明原理） |

---

## ステップ1: 引数の解析

引数の先頭が「使えるスキル」テーブルのキーと一致するか確認する。

- **`/think <スキル名> <入力>`**: スキルを明示指定。`<入力>` 部分を `{input}` とし、ステップ3の該当スキルへ直接委譲する
  - `<入力>` が空の場合: 「何を分析しますか?」と聞き、返答を `{input}` とする
  - スキル名がテーブルに存在しない場合: 「スキル名 `<X>` は存在しません。使えるスキル: six-hats / scamper / first-principles / triz」と返してSTOP
- **`/think <入力>`**: スキル名なし。`<入力>` 全体を `{input}` としてステップ2に進む
- **`/think`（引数なし）**: 「何を考えたいですか?」と一言聞く。返答を `{input}` としてステップ2に進む

---

## ステップ2: 入力の自動分類

以下の基準で `{input}` を判定する。

### six-hats向け（全てを満たす場合）

1. 具体的な提案・計画・選択肢が含まれている
2. 多角的な視点での検証・論点整理が目的
3. 「答えを1つ出してほしい」ではなく「検証・整理」を求めている

### scamper向け（全てを満たす場合）

1. 変形・改良・発展させる「既存の対象」が含まれている
2. 「もっと良くしたい」「別の展開はないか」という発展探索が目的

### first-principles向け（全てを満たす場合）

1. 「なぜそうなのか」「本当にそうなのか」という根本的な疑問を含んでいる
2. 業界の常識・慣習を疑い、ゼロから考え直すことが目的

### triz向け（全てを満たす場合）

1. 「AをよくするとBが悪化する」「〇〇でありながら〇〇でもある」という矛盾・トレードオフを含んでいる
2. 妥協ではなく矛盾そのものを解消したい

### 未分類（上記のいずれにも該当しない場合）

---

## ステップ3: 委譲または確認

### スキル名を明示指定した場合（ステップ1で直接委譲）

確認なしに即座に実行する。

- six-hats: `{SKILL_DIR}/six-hats/SKILL.md` を READ し実行
- scamper: `{SKILL_DIR}/scamper/SKILL.md` を READ し実行
- first-principles: `{SKILL_DIR}/first-principles/SKILL.md` を READ し実行
- triz: `{SKILL_DIR}/triz/SKILL.md` を READ し実行

### 自動判定した場合（ステップ2を経由）

以下の形式でスキル選択の理由を示し、yes/no を確認する。

```
スキル: <スキル名>
理由: <判定基準のどの条件に該当したか、1〜2文で具体的に>

このスキルで進めますか？ [yes / no / <別スキル名>]
```

WAIT_FOR: ユーザーの返答
- `yes` または Enter → 選択スキルの SKILL.md を READ して実行する
- `no` → 「使えるスキル」テーブルを表示してスキルを選ばせる
- `<別スキル名>` → そのスキルの SKILL.md を READ して実行する

### 未分類の場合

「使えるスキル」テーブルを参照し、以下を表示してユーザーに選択を求める。

```
どのスキルを使いますか？

現在使えるスキル:
- six-hats: 具体的な提案・計画・選択肢の多角的検証
- scamper: 既存アイデア・製品・プロセスの変形・発展
- first-principles: 常識・慣習を疑ってゼロから再構築
- triz: 相反する要件を同時に満たす解法を探す

スキルを使わず直接回答することもできます。
```

WAIT_FOR: ユーザーの選択
- スキルを選んだ場合 → そのスキルの SKILL.md を READ して実行する
- 直接回答を選んだ場合 → `{input}` に対して直接回答する
