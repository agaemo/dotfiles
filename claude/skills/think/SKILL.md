# think — 思考・分析オーケストレーター

思考・分析・検討の相談を受け付けるオーケストレーター。
「〜を分析したい」「〜について考えたい」「six-hats で検証したい」のような
入力に対して適切なスキルへ委譲する。

```
SKILL_DIR = スキル起動時に提供される "Base directory for this skill:" のパス
```

## 使えるスキル

| スキル | 得意な入力 |
|-------|-----------|
| six-hats | 具体的な提案・計画・選択肢の多角的検証（Six Thinking Hats） |

---

## ステップ1: 引数の解析

引数の先頭が「使えるスキル」テーブルのキーと一致するか確認する。

- **`/think <スキル名> <入力>`**: スキルを明示指定。`<入力>` 部分を `{input}` とし、ステップ3の該当スキルへ直接委譲する
  - `<入力>` が空の場合: 「何を分析しますか?」と聞き、返答を `{input}` とする
  - スキル名がテーブルに存在しない場合: 「スキル名 `<X>` は存在しません。使えるスキル: six-hats」と返してSTOP
- **`/think <入力>`**: スキル名なし。`<入力>` 全体を `{input}` としてステップ2に進む
- **`/think`（引数なし）**: 「何を考えたいですか?」と一言聞く。返答を `{input}` としてステップ2に進む

---

## ステップ2: 入力の自動分類

以下の基準で `{input}` を判定する。

### six-hats向け（全てを満たす場合）

1. 具体的な提案・計画・選択肢が含まれている
2. 多角的な視点での検証・論点整理が目的
3. 「答えを1つ出してほしい」ではなく「検証・整理」を求めている

### 未分類（上記に該当しない場合）

---

## ステップ3: 委譲または確認

### six-hats の場合

「six-hats で分析します。」と一言伝えてから、
`{SKILL_DIR}/six-hats/SKILL.md` を READ し、ファイル内の `{input}` を分析対象テキストに置換した上でフロー全体を実行する。

### 未分類の場合

「使えるスキル」テーブルを参照し、以下を表示してユーザーに選択を求める。

```
どのスキルを使いますか？

現在使えるスキル:
- six-hats: 具体的な提案・計画・選択肢の多角的検証

スキルを使わず直接回答することもできます。
```

WAIT_FOR: ユーザーの選択
- スキルを選んだ場合 → そのスキルの SKILL.md を READ して実行する
- 直接回答を選んだ場合 → `{input}` に対して直接回答する
