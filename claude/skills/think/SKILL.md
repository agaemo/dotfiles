# think — 思考・分析オーケストレーター

問いや課題を受け取り、**intent**（発散/収束）と **verbosity**（ひと言/箇条書き/フル論拠付き）を判定して
サブスキルに委譲する。

```
SKILL_DIR = スキル起動時に提供される "Base directory for this skill:" のパス
```

## サブスキル一覧

| intent | スキル | 説明 |
|---|---|---|
| 発散 | ideate | 課題・問いから具体的な提案・アイデアを生成 |
| 発散 | scamper | 既存の対象を7軸で変形・発展（SCAMPER法） |
| 収束 | six-hats | 提案・計画・選択肢を6視点で多角的検証 |
| 収束 | triz | 矛盾・トレードオフを40発明原理で解消 |
| 収束 | first-principles | 常識・慣習を疑いゼロから再構築 |
| 収束 | wardley | 事業・製品・システムをバリューチェーン×進化軸で可視化し戦略的判断を導く |
| 対話 | dialogue | 答えがまだ定まっていない曖昧な相談に、問いを重ねて壁打ちする |

---

## ステップ1: 引数の解析

引数の先頭がサブスキル名と一致するか確認する。

- **`/think <スキル名> <入力>`**: スキルを明示指定。`<入力>` を `{input}` とし、ステップ3の該当スキルへ直接委譲する
  - `<入力>` が空 → ここで汎用質問はしない。`{input}` = "" のままステップ3に進み、委譲先スキルの「引数なし」分岐（各スキルが用意する専用の質問文）に判断を委ねる
  - スキル名が存在しない → 「スキル名 `<X>` は存在しません。使えるスキル: ideate / scamper / six-hats / triz / first-principles / wardley / dialogue」と返してSTOP
- **`/think <入力>`**: スキル名なし。`<入力>` 全体を `{input}` としてステップ2に進む
- **`/think`（引数なし）**: 以下を表示して WAIT_FOR: ユーザーの返答

  ```
  何を壁打ちしたいですか？

  今どの段階ですか:
  - まだモヤモヤ（何から考えればいい？）
  - 方向性はある（深めたい）
  - 案はある（検証・評価したい）
  ```

  返答を `{input}` とし、選択した段階を intent 判定のヒントとしてステップ2に進む（モヤモヤ→発散寄り / 方向性はある→ステップ2の通常判定に委ねる / 案はある→収束寄り）

## ステップ2: intent と verbosity の判定

### verbosity の判定

`{input}` に以下が含まれる場合:
- 「ひと言」「さっと」「手短に」「手軽に」→ `{verbosity_label}` = ひと言、`{verbosity}` = 簡潔
- 「詳しく」「丁寧に」「しっかり」「論拠付き」「根拠つき」→ `{verbosity_label}` = フル論拠付き、`{verbosity}` = 詳細
- それ以外 → `{verbosity_label}` = 箇条書き、`{verbosity}` = 標準

**注**: サブスキルには `{verbosity}`（簡潔/標準/詳細）を渡す。`{verbosity_label}` はユーザー表示専用。

### intent の判定

**発散 / ideate（以下を満たす場合）**
- 「アイデアを出してほしい」「何かないか」「案を考えてほしい」「提案してほしい」
- 新しいアイデア・解決策の生成が目的
- ただし変形する既存の対象が明確な場合は scamper を優先する

**発散 / scamper（以下を満たす場合）**
- 変形・改良する「既存の対象」が明確に含まれている
- 「もっと良くしたい」「別の展開はないか」という発展探索が目的

**収束 / six-hats（以下を満たす場合）**
- 具体的な提案・計画・選択肢が含まれている
- 「検証したい」「評価したい」「リスクを洗い出したい」「良し悪しを整理したい」が目的

**収束 / triz（以下を満たす場合）**
- 「〇〇するとXが悪化する」「両立できない」「相反する要件がある」という矛盾・トレードオフを含む
- 妥協でなく矛盾そのものを解消したい

**収束 / first-principles（以下を満たす場合）**
- 「なぜ〇〇なのか」「本当にそうか」という根本的な疑問がある
- 業界の常識・慣習を疑いゼロから考え直したい

**収束 / wardley（以下を満たす場合）**
- 「内製すべきか外注すべきか」「どこに投資するか」という戦略的判断が目的
- 既存の事業・製品・システムのコンポーネントを可視化・整理したい
- 競合との差別化ポイントや投資優先度を把握したい

**対話 / dialogue（以下を満たす場合）**
- 本人の中で答え・方向性・目標がまだ定まっていない曖昧な相談
- 「なんとなくモヤモヤする」「頭の中を整理したい」が目的で、発散（アイデアが欲しい）でも収束（案の検証がしたい）でもない
- 事実質問・比較質問（答えが一意に定まるもの）はここに該当しない → 直接回答する

## ステップ3: 委譲

### スキル名を明示指定した場合（ステップ1で直接委譲）

確認なしに即座に実行する。`{input}` が空でない場合はステップ2の verbosity 判定ロジックに準じて `{input}` から判定する。`{input}` が空の場合は一旦 `{verbosity}` = 標準 とし、委譲先スキルが「引数なし」分岐で実際の入力を取得した時点で、その入力に対してステップ2の verbosity 判定ロジックを再適用する。

選択スキルの `{SKILL_DIR}/<スキル名>/SKILL.md` を READ し、`{input}` と `{verbosity}` を渡して実行する。
SKILL.md が存在しない・READ 失敗の場合は「スキル `<X>` を起動できません」と伝えてSTOP。

### 自動判定した場合（ステップ2を経由）

`[発散]` `[収束]` または `[対話]` ＋ 説明1文の形式で提示し、確認する:

```
[発散] アイデアを広げ具体的な提案を生成します
手法: ideate（Brown 2008 / Christensen 2016）
粒度: 箇条書き

このまま進めますか？ [yes / 別の方向で / 直接回答して / <別スキル名>]
```

`[対話]`（dialogue）の場合は verbosity を扱わないため、粒度の行は表示しない:

```
[対話] 答えを急がず、問いを重ねながら一緒に整理します
手法: dialogue（Paul & Elder, 1990s / Whitmore, 1992）

このまま進めますか？ [yes / 別の方向で / 直接回答して / <別スキル名>]
```

論拠の例: ideate → Brown (2008) / Christensen (2016)、scamper → Eberle (1971) / Osborn (1953)、six-hats → de Bono (1985)、triz → Altshuller (1956)、first-principles → Aristotle、wardley → Wardley (2005)、dialogue → Paul & Elder (1990s) / Whitmore (1992)

WAIT_FOR: ユーザーの返答
- `yes` または Enter → 選択スキルの SKILL.md を READ し、`{input}` と `{verbosity}` を渡して実行する
- `別の方向で` / `no` → サブスキル一覧を表示してスキルを選ばせる
- `直接回答して` → フレームを使わず `{input}` に対して直接回答する
- `<別スキル名>` → サブスキル一覧に存在するか確認する。存在しない場合は「スキル名 `<X>` は存在しません」と伝えて再度選択を促す。存在する場合はその SKILL.md を READ し、`{input}` と `{verbosity}` を渡して実行する

### 未分類の場合

いずれにも該当しない場合は以下を表示してユーザーに選択を求める。

```
どのスキルを使いますか？

発散（アイデア・提案を出す）:
- ideate: 課題から具体的な提案を生成
- scamper: 既存のアイデア・製品・プロセスを変形・発展

収束（検証・整理・解消する）:
- six-hats: 提案・計画を多角的に検証
- triz: 矛盾・トレードオフを解消
- first-principles: 常識を疑いゼロから再構築
- wardley: 事業・製品をバリューチェーン×進化軸で可視化し戦略を導く

対話（答えがまだない曖昧な相談）:
- dialogue: 問いを重ねながら一緒に整理する

スキルを使わず直接回答することもできます。
```

WAIT_FOR: ユーザーの選択
- スキルを選んだ場合 → そのスキルの SKILL.md を READ し、`{input}` と `{verbosity}` を渡して実行する
- 直接回答を選んだ場合 → `{input}` に対して直接回答する

## 連鎖モード

サブスキルの実行が完了した後、対話中にいつでも以下のコマンドで別スキルへ連鎖できる:

```
→ <スキル名>
```

例:
- `→ six-hats` — 出たアイデアをリスク・実現性の観点で検証する
- `→ triz` — 見えてきた矛盾・トレードオフを解消する
- `→ first-principles` — 前提を疑いゼロから再検討する
- `→ ideate` — 評価フェーズから発想フェーズに戻る
- `→ wardley` — 事業・製品の構造を戦略マップで整理する
- `→ dialogue` — 答えを急がず、問いを重ねながら整理し直す

連鎖時は直前の `{input}` と会話文脈を引き継ぎ、`{verbosity}` と `{verbosity_label}` はそのまま維持する。
dialogue から他スキルへ連鎖する場合、dialogue は `{verbosity}` を保持していないため `{verbosity}` = 標準 とする。
スキル名が存在しない場合は「スキル名 `<X>` は存在しません。使えるスキル: ideate / scamper / six-hats / triz / first-principles / wardley / dialogue」と返して連鎖を中断する。
