# 親子スキル一貫性チェック（CROSS_MODE = true のみ）

```
REQUIRE: MULTI_FILE_PROCESS の全ファイル処理（ステップ2〜4）完了後に実行する
NOTE: 各ファイルの EXTRACT 結果をそのまま使う。ファイルの再読みは行わない

COMPARE 親スキル vs 各子スキル:

[X1] 呼び出しインターフェース
  CHECK: 親スキルが子スキルを呼び出す際に渡す引数・プロンプトと、
         子スキルが期待する入力（引数・変数・前提条件）が一致しているか

[X2] キーワード統一
  CHECK: GATE / WAIT_FOR / PROHIBITED / REQUIRE 等の表記が
         親子間で統一されているか

[X3] 出力フォーマット慣習
  CHECK: 親が子の出力を受け取る場合、子の出力フォーマットと
         親が期待するフォーマットが一致しているか

[X4] エラー・異常系の引き継ぎ
  CHECK: 子スキルが失敗・中断した場合の処理が親スキル側に定義されているか

GENERATE 改善提案（不一致がある場合のみ）:
  分類: [X1〜X4]
  優先度: HIGH（動作に影響）/ MEDIUM（品質に影響）
  場所: 親スキル or 子スキル（修正すべき側）
  問題: <不一致の内容>
  推奨: <修正案>
  理由: <1行>

不一致がない場合: "親子スキル一貫性: 問題なし" とのみ報告する
```
